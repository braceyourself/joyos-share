Connect Your Stripe Account

Link Stripe to the JoyOS Lab platform.

Hey Kristin

I know you've done Stripe before, so most of this will be familiar. The two things that are different from a normal setup:

  • A restricted API key — instead of your main secret key, we create one that only has the permissions the platform needs. Safer that way.
  • A webhook — this is a URL that Stripe pings every time someone pays, cancels, or changes their subscription. Without it, the platform won't know when someone has paid.

If you already have your Activator and Catalyst products in Stripe, skip the product creation steps. The whole thing should take about 5 minutes.

Log in to Stripe

Go to dashboard.stripe.com and sign in with your Stripe account.

If you don't have a Stripe account yet, you can create one there. You'll need a bank account connected for payouts.

Create Your Products

You need two subscription products in Stripe. If you already have them, skip this step.

  1. Go to Product catalog in the left sidebar
  2. Click + Add product
  3. First product:
    • Name: Joy Activator
    • Pricing: Recurring, $297.00 per year
  4. Second product:
    • Name: Joy Catalyst
    • Pricing: Recurring, $597.00 per year
You don't need Joy Seeker yet. We're keeping that tier hidden until the platform is more built out.

Go to API Keys

  1. Click Developers in the top-right corner of the Stripe dashboard
  2. Click API keys in the left sidebar

You'll see your existing keys. We're going to create a new restricted key that only has the permissions the platform needs.

Create a Restricted Key

Click + Create restricted key, name it JoyOS Lab Platform, and set these permissions:

PermissionLevel
Checkout SessionsWrite
CustomersWrite
Customer portalWrite
ProductsRead
PricesRead
SubscriptionsRead

Everything else stays None. Click Create key.

Stripe only shows the full key once. It starts with rk_live_. Copy it before closing the dialog.

Paste the Key into the Admin Panel

  1. Go to lab.joyos.global/app/admin
  2. Sign in with your Google account
  3. In the Stripe tab, paste the key into the Restricted API Key field
  4. Don't click Save yet, we'll do that after the webhook step
Your key is stored securely and only used server-side. It never appears in the browser or in any code that members can see.

Set Up the Webhook

This tells Stripe to notify the platform when someone pays, cancels, or changes their subscription.

  1. In the Stripe dashboard, click Developers (top-right)
  2. Click Webhooks in the left sidebar
  3. Click + Add endpoint
  4. For the endpoint URL, paste:
    https://lab.joyos.global/api/stripe-webhook
  5. Click Select events, then check these five:
    • checkout.session.completed
    • customer.subscription.updated
    • customer.subscription.deleted
    • invoice.paid
    • invoice.payment_failed
  6. Click Add endpoint
You only need to do this once. After it's set up, every payment and subscription change will automatically update the member's access on the platform.

Paste the Webhook Signing Secret

After creating the webhook, Stripe will show you a Signing secret that starts with whsec_.

  1. Copy the signing secret
  2. Go back to the admin panel Stripe tab
  3. Paste it into the Webhook Signing Secret field
  4. Click Save

This lets the platform verify that webhook events are actually coming from Stripe.

Stripe only shows this secret once. Copy it before navigating away from the page.

Everything is connected. When someone pays on the platform, Stripe processes the payment and the platform updates their access automatically.
